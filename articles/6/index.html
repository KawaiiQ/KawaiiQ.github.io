Coursera《机器学习》学习笔记（六）：推荐系统篇

2018-02-03

这一篇是《机器学习》第八周到第九周后半部分知识点的总结，介绍了机器学习的一个应用的方向：推荐系统。推荐系统在很多地方都有应用，淘宝京东的推荐商品、电影网站的推荐电影、网易云音乐的的推荐音乐,我们几乎到处都能看到推荐系统的影子。通过推荐系统，厂商可以更好地提供个性化服务，提升用户体验，从而从你口袋里掏出更多的钱。

主要内容

  • 推荐系统要干什么
  • 基于内容的推荐
  • 协同过滤
  • 低秩矩阵分解


推荐系统要干什么

引用课程中个电影评分网站的例子，假如我们有一个电影网站，用户可以给每部电影打分（0~5颗星），现在我们有5部电影和4名用户的评分情况，如下，“-”为没有打分。

电影 Alice Bob Carol Dave
《爱到最后》 5 5 0 0
《永远的浪漫》 5 - - 0
《小爱犬》 - 4 0 -
《无尽狂飙》 0 0 5 4
《剑 vs. 空手道》 0 0 5 -

我们这里不去分析用户名和电影名之中可能包含的信息，所以这里用户名和电影名都无所谓，在计算机中实际上是对用户和电影进行编号的，如下。

电影 (1) (2) (3) (4)
(1) 5 5 0 0
(2) 5 - - 0
(3) - 4 0 -
(4) 0 0 5 4
(5) 0 0 5 -

为了便于之后形式化地描述和解决推荐系统的问题，这里约定一些符号。

  • ：用户的数量
  • ：电影数量
  • ：用户评价过的电影数量
  • ：用户是否是否评价了电影，表示已评价，表示未评价。
  • ：用户对电影的评分。

而我们要解决的问题，就是对于用户，预测该用户对的电影可能的评分。

例如上面的例子，我们直观地看电影标题的话，前三个电影大概是爱情电影，后俩更像是动作片。再结合用户的评分，我们不难看出，Alice和Bob喜欢爱情电影，而Carol和Dave喜欢动作电影。再根据电影题材，我们就能够大致推测出他们对于没打分的电影可能的打分了。这是我们人进行推理的过程，而计算机所要做的其实也是和这差不多，只不过其并不能一眼“看”出来电影的类型和用户的喜好，而是要通过特征进行训练。

当通过预测得到用户对于所有电影的打分后，我们就能根据评分来针对性地推荐该用户没看过的电影给他（她）了。


基于内容的推荐系统

最简单的思路就是，根据电影内容得到电影的一些特征，再加上用户的打分作为训练集，通过训练拟合每个用户的打分策略，即用户对不同电影的偏好，来预测每个用户对于他（她）没有看过的电影可能会打多少分。

均值归一化

等号右边第一项由于用户没有打过分，所以是，由于包含电影特征项的正则项是训练好的，所以是不变的，那么为了得到最小值，显然新用户的。然而这样并没有啥意义，因为我们还是不知道给他（她）推荐啥……

对此的解决方法是使用均值归一化，对于每部电影，我们计算所有该电影打的分的均值，也就是给定，求满足的所有，求所有的均值。接下来，对于所有的，均令其减去，然后再进行训练。进行预测时，在计算得到预测值后再加上作为最终预测值。

这样做的话，对于一个全新的用户，依然是，但是最后计算预测值时，由于加上了均值，最后得到的，就是这部电影被打的所有分的均值。这样，我们就能够推荐平均分最高的那部电影了。


总结

推荐系统算是线性回归算法的一个应用，总体来看，推荐系统训练的思想和K均值算法很像，都是交替进行，固定一类参数优化另一类，不过推荐系统在此基础上更进了一步，实现了两者的同时更新。
